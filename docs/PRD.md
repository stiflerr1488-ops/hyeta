# PRD — Визуальный редактор многостраничного HTML-сайта (MVP)

## Цель
Сократить время внесения типовых правок в многостраничный сайт без сборки: тексты, CTA, ссылки, контакты, изображения и базовые стили. Целевой результат — пользователь открывает локальный проект, редактирует визуально, проверяет в живом превью, сохраняет и экспортирует ZIP для загрузки на хостинг.

## Пользователи
- Основной: владелец/маркетолог (базовый HTML/CSS, минимум кода).
- Дополнительный: разработчик/верстальщик (точечные CSS/JS-правки).

## Основные сценарии
1. Импорт проекта из локальной директории.
2. Навигация по дереву файлов и списку HTML-страниц.
3. Открытие страницы в живом превью (с JS) с поддержкой относительных путей.
4. Клик по элементу в превью и правка текста/атрибутов (href/src/alt/title).
5. Отмена/повтор в рамках сессии.
6. Поиск по проекту.
7. Сохранение правок в исходные файлы и экспорт проекта в ZIP.

## Ключевые функции MVP
- Подход «сначала десктоп», «сначала Windows», «офлайн по умолчанию».
- Индексация проекта: .html/.css/.js/.svg/.png/.webp/.ico/.xml/.webmanifest/.json/.txt/.php (как текст).
- Разделённый интерфейс: слева дерево файлов, по центру превью, справа панель свойств.
- Встроенное редактирование через выбор элемента в превью.
- История изменений на уровне файла (отмена/повтор).
- Поиск по проекту и выдача по строкам.
- Сохранение изменений в файлы.

## Ограничения MVP
- Нет конструктора блоков с перетаскиванием.
- Нет совместного редактирования в реальном времени и ролей.
- Нет ИИ-изменений.
- Нет встроенной публикации в SFTP/FTP (после MVP).

## Метрики успеха
- Время до правки: путь «нашёл → изменил → сохранил» заметно быстрее ручного редактирования.
- Корректность: относительные пути не ломаются.
- Производительность: быстрый старт и отзывчивые операции на проектах до 500–1500 файлов.
- Безопасность: песочница превью и блок внешней сети по умолчанию.
