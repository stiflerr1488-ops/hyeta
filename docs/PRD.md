# PRD — Визуальный редактор многостраничного HTML-сайта (MVP)

## Цель
Сократить время внесения типовых правок в мультистраничный сайт без сборки: тексты, CTA, ссылки, контакты, изображения и базовые стили. Целевой результат — пользователь открывает локальный проект, редактирует визуально, проверяет в живом превью, сохраняет и экспортирует ZIP для загрузки на хостинг.

## Пользователи
- Primary: владелец/маркетолог (базовый HTML/CSS, минимум кода).
- Secondary: разработчик/верстальщик (точечные CSS/JS-правки).

## Основные сценарии
1. Импорт проекта из локальной директории.
2. Навигация по дереву файлов и списку HTML-страниц.
3. Открытие страницы в живом превью (с JS) с поддержкой относительных путей.
4. Клик по элементу в превью и правка текста/атрибутов (href/src/alt/title).
5. Undo/Redo в рамках сессии.
6. Поиск по проекту.
7. Сохранение правок в исходные файлы и экспорт проекта в ZIP.

## Ключевые функции MVP
- Desktop-first, Windows-first, offline-by-default.
- Индексация проекта: .html/.css/.js/.svg/.png/.webp/.ico/.xml/.webmanifest/.json/.txt/.php (как текст).
- Split-view: слева дерево файлов, центр preview, справа панель свойств.
- Inline редактирование через выбор элемента в preview.
- История изменений на уровне файла (undo/redo).
- Поиск по проекту и выдача по строкам.
- Сохранение изменений в файлы.

## Ограничения MVP
- Нет drag-and-drop конструктора блоков.
- Нет real-time коллаборации и ролей.
- Нет AI-изменений.
- Нет встроенного deploy в SFTP/FTP (после MVP).

## Метрики успеха
- Time-to-edit: путь «нашел → изменил → сохранил» заметно быстрее ручного редактирования.
- Корректность: относительные пути не ломаются.
- Производительность: быстрый старт и отзывчивые операции на проектах до 500–1500 файлов.
- Безопасность: sandbox preview и блок внешней сети по умолчанию.
